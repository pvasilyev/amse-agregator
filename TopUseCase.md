## User stories ##
С помощью одной из методик разработки По, были проанализированы некоторые социальные группы.

1. **Вася программист**
Вася веселый молодой парень, который хорошо зарабатывает. Едет в Европу, чтобы отдохнуть с друзьями, такими же молодыми и веселыми ребятами. Главная цель это молодежный отдых: с утра прогулки по городу, днем пляж, бассейн, вечером-ночью клубы (и, возможно, казино). Его также интересует shopping, места для проведения вечеринок и прочее.

2. **Бизнесмен Георгий**
Георгий едет в Европу для деловой поездки, он крупный бизнесмен. Он должен четко знать, когда и куда он может сходить в свое свободное время (расписание для него самое важное). В каких ресторанах он может обсуждать деловые вопросы со своими колллегами. Возможно его будет интересовать прокат авто/велосипеда/еще-чего-нибудь для независимого передвижения. Если он будет передвигаться общественным транспортом, то его будет интересовать время в пути и точный маршрут.

3. **Молодая семья: Дима + Катя + маленький сынуля Ваня**
Эта молодая семья немного ограничена в бюджете, поэтому будет важна информация о ценах. Также будет интересно место проживания и вопрос удобства передвижения с маленьким сыном. После "культурного минимума" маленький Ваня может устать и для него можно подобрать что-то детское-играющее: аквапарк, карусельки, пляж или детские игровые автоматы.

4. **Супружеская пара Василий Афанасьевич и Татьяна Петровна**
Супружеская пара в возрасте едет на какой-нибудь курорт или в санаторий. Хотят узнать побольше об этой стране. Будут интересны культовые места, национальные особеннобенности и сувениры, shopping-центры. Для них крайне важен вопрос передвижения, поскольку они довольно пожилые.

# Top use cases #

INCLUDED:

  * **Поиск по городам**. User вводит в строку запроса название города (например, "_милан_"). Результат запроса -- это описание города и список самых релевантных достопримечательностей. Подробную информацию с описанием можно найти для кажой достопримечательности. (подробнее см. "поиск по достопримечательностям"). Далее к списку объектов можно применять фильтрацию согласно полям сущности (например, "время постройки" >= 17 век и т.д.).

  * **Поиск по достопримечательностям**. User вводит в строку запроса известную достопримечательность (наприрмер, "_колизей_" или "_пизанская башня_") или ключевые слова (например, "_архитектура_" или "_живопись_"). В зависимости от этого результатом может быть конкретный объект (в случае точного запроса) или список объектов, которые попадают под описание запроса (если запросом была строка "_милан архитектура_", то его результатом будет список архитектурных достопримечательностей, расположенных в милане). С помощью feature "ближайшие" или "related" можно найти релевантные объекты, которые находятся рядом с выбранным или похожи по тематике (ключевым словам). Подробнее см. "поиск по ближайшим". Также можно пометить данную достопримечательность как "я хочу посетить" или "добавить в список для составления маршрута". Подробнее см. "составить маршрут".

  * **Поиск по ближайшим** или "related". Наподобие "related videos" в youtube можно находить related и ближайшие достопримечательности, отранжированные по релевантности.

  * **Составить маршрут**. После того, как в условной "корзине" собирается несколько объектов для посещения, user может составить маршрут с помощью этой feature. На карте отмечаются маршруты и возможный транспорт, с помощью которого можно добраться до следующей достопримечательности.

  * **Поиск по клубам, ресторанам, отелям, аптекам, ...**. Поиск по тематическим объектам можно осуществить с помощью связи с другим проектом.

  * **Изучение отзывов, комментариев**. User может просматривать и оставлять комментарии о том или ином объекте с целью узнать "народное" мнение. В понятие "_отзыва_" включается информация из интернета (которую достал miner), а в понятие "_комментария_" включается пост user'а, который можно оставить на нашем сайте.

  * **Организация экскурсии / гида**. Мало, кто из туристов знает, что в Петербурге можно заказать метеор от "Авроры" до фонтанов "Петродворца". Это редкая информация. User может искать экскурсии, которые будут охватывать те или иные объекты. Usability большой: многие предпочитают охватить несколько объектов за один раз, чтобы сэкономить время в очередях.

  * **Изучение дополнительной информации**. Под дополнительной информацией подразумевается:
    * Памятки туристу (что можно делать в этой стране или городе и чего нельзя делать, а также, к кому обратиться за помощью). Например, чтобы поехать в африканскую страну, нужно сделать прививку от малярии. Это знают не все.
    * Особенности данного района. Погодные, климатические, криминогенные и прочие особенности. Например, user может ввести дату своего пребывания в данном городе, а поисковая система скажет: "в этот день по городу будут пускать быков -- это народная забава в Испании; поэтому воздержитесь от прогулок по центру города".
    * Вывести рядом на карте важные места: аптеки, банкоматы, магазины.

NOT INCLUDED:

  * **Поиск по гостиницам**. Поиск по гостиницам можно осуществлять на более мощных web-ресурсах, таких как http://www.hostelsworld.com/ или http://www.bookings.com/. Это специализированные сайты с конкретной целевой задачей. Можно найти гостиницу или хостел и забронировать номер на определенную дату с предоплатой или без.
